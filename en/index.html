A single annotation transforms any Spring Boot method into a structured business event. No agents. No infra changes. Just data your team actually uses.

Install in 2 minutes →⭐ View on GitHub
PedidoController.java● live
@EchoTrace(name = "pedido.request",
         capture={"pedido.id"}, captureReturn = true)
public PedidoResponse criarPedido(@RequestBody PedidoRequest pedido) {

  // Enrich the event at any point in the flow
  Telemetry.capture("valor_gasto", pedido.getValor());

  PedidoResponse response = new PedidoResponse();
  Telemetry.capture("expectativa_entrega", response.getHorarioExpectativaEntrega());

  return response;
}

The problem

You see the CPU.
You don't see the business.

✗ Without EchoTrace
  • CPU at 12%, memory ok — but are orders completing?
  • Stable p99 latency — where are users dropping off?

✓ With EchoTrace
  • Every order attempt emits a structured event with full context.
  • Journey steps captured as named business events.
  • Real outcome — success, partial, failure — in every payload.
  • Trace ID propagated across the entire microservice graph.

Get started

Up and running in 4 steps

1
Add the Maven dependency
pom.xml
<dependency>
  <groupId>io.echotrace</groupId>
  <artifactId>echotrace-spring-boot-starter</artifactId>
  <version>1.0.0</version>
</dependency>
2
Annotate your first method
OrderController.java
# Annotate any method — that's it. Events are delivered automatically.
$ @EchoTrace(name = "order-create", capture="req.id", captureReturn = true)
$ public OrderResponse createOrder(OrderRequest req) { ... }
✓ Event emitted: { "event": "order.create", ... }
3
Configure the HTTP target
application.properties
# Define the collector URL for automatic payload forwarding
echotrace.collector-url=http://localhost:8080
4
Ready!!
{
  "event":   "order-create",
  "class":   "OrderController",
