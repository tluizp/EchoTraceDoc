Uma anotação transforma qualquer método Spring Boot em um evento de negócio estruturado. Sem agentes. Sem mudanças de infra. Só dados que o seu time realmente usa.

Instalar em 2 minutos →⭐ Ver no GitHub
PedidoController.java● live
@EchoTrace(name = "pedido.request",
         capture={"pedido.id"}, captureReturn = true)
public PedidoResponse criarPedido(@RequestBody PedidoRequest pedido) {

  // Enriqueça o evento em qualquer ponto do fluxo
  Telemetry.capture("valor_gasto", pedido.getValor());

  PedidoResponse response = new PedidoResponse();
  Telemetry.capture("expectativa_entrega", response.getHorarioExpectativaEntrega());

  return response;
}

Get started

Funcionando em 4 passos

1
Adicione a dependência Maven
pom.xml
<dependency>
  <groupId>io.echotrace</groupId>
  <artifactId>echotrace-spring-boot-starter</artifactId>
  <version>1.0.0</version>
</dependency>
2
Anote seu primeiro método
OrderController.java
# Anote qualquer método — pronto. Eventos chegam automaticamente.
$ @EchoTrace(name = "order-create", capture="req.id", captureReturn = true)
$ public OrderResponse createOrder(OrderRequest req) { ... }
✓ Evento emitido: { "event": "order.create", ... }
3
Configure o destino HTTP
application.properties
# Defina a URL do collector para o envio automático de payloads
echotrace.collector-url=http://localhost:8080
4
Pronto!!
{
  "event":   "order-create",
  "class":   "OrderController",
  "req.id": "A12VDf4SDfds5",
  "methodReturn": "OrderResponse(id=A12VDf4SDfds5, status=EM ANDAMENTO)",
  "expectativa_entrega": "19:30",
  "durationMs": 78
}
// Seu sistema já gera os dados.
